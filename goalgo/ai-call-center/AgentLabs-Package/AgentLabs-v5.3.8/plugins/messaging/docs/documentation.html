Overview

The Messaging Plugin adds email and WhatsApp messaging capabilities to your AgentLabs AI agents. During or after voice calls, agents can send emails via SMTP and WhatsApp messages through your choice of provider — giving your customers a seamless multi-channel experience.

Quick Start Guide

Email — 5-Minute Setup

  1. Go to Settings → Messaging in your user dashboard
  2. Click the Email Templates tab
  3. Create a new template with your SMTP server details (host, port, username, password)
  4. Write your email subject and body (supports HTML and the Unlayer drag-and-drop editor)
  5. Click Send Test Email to verify your SMTP connection
  6. Go to your agent settings, enable Email Sending under System Tools, and select your template

WhatsWay WhatsApp — 5-Minute Setup

  1. Sign up at whatsway.com and get your API Key and API Secret
  2. Go to Settings → Messaging → WhatsApp tab
  3. Enter your WhatsWay API Key and API Secret
  4. Click Test Connection to verify
  5. Go to your agent settings, enable WhatsApp Sending under System Tools

Meta WhatsApp Business API — Setup

  1. Ask your admin to configure Meta App credentials in Admin Panel → Settings → Messaging
  2. Go to Settings → Messaging → WhatsApp tab
  3. Click Connect with Facebook to start the Embedded Signup flow
  4. Follow the Meta prompts to connect your WhatsApp Business Account
  5. Your WABA ID, Phone Number ID, and Access Token are saved automatically
  6. Go to your agent settings, enable WhatsApp Sending under System Tools

Installation

The Messaging Plugin integrates with the AgentLabs plugin system. Choose one of the methods below.

Method 1: Admin UI Installer (Recommended)

Upload Plugin ZIP

Navigate to Admin Panel → Settings → Plugins. Click Upload Plugin and select the messaging-plugin.zip file.

Automatic Setup

The installer will validate the plugin manifest, extract files to plugins/messaging/, and run database migrations automatically.

Restart Server

Restart the AgentLabs server to load the plugin. The Messaging tab will appear in both Admin Settings and User Settings.

Configure Provider

Go to Admin Panel → Settings → Messaging to set the WhatsApp provider mode and Meta App credentials.

Method 2: Manual Installation (SSH)

Extract Plugin Files

# Extract to the plugins directory
cd /path/to/agentlabs
unzip messaging-plugin.zip -d plugins/messaging/

Install Dependencies

# Install peer dependencies (if not already present)
npm install nodemailer multer
npm install --save-dev @types/nodemailer @types/multer

Run Database Migrations

# Run migrations in order
psql -d agentlabs -f plugins/messaging/migrations/001_messaging_tables.sql
psql -d agentlabs -f plugins/messaging/migrations/002_meta_whatsapp.sql
psql -d agentlabs -f plugins/messaging/migrations/003_whatsapp_conversations.sql
psql -d agentlabs -f plugins/messaging/migrations/004_logs_content.sql

Register Plugin

The plugin is automatically discovered via its plugin.json manifest. Restart the server:

pm2 restart agentlabs
# or
systemctl restart agentlabs

Requirements

  • AgentLabs v1.0.0 or later
  • Node.js 18+
  • PostgreSQL database
  • npm packages: express, drizzle-orm, nodemailer, multer

Email Setup

The email channel uses SMTP to deliver messages. Each user creates their own email templates with embedded SMTP credentials.

SMTP Configuration

When creating an email template, you provide the SMTP server details:

Field Description Example
smtpHost SMTP server hostname smtp.gmail.com
smtpPort SMTP server port 587 (TLS) or 465 (SSL)
smtpUser SMTP username / email you@gmail.com
smtpPass SMTP password or app password xxxx-xxxx-xxxx-xxxx
fromName Sender display name My Company
fromEmail Sender email address noreply@mycompany.com
ℹ
Gmail Users: You must use an App Password instead of your regular Gmail password. Enable 2-Step Verification first, then generate an App Password.

Email Templates

Each email template includes:

  • Template Name — A descriptive name for identification
  • Subject Line — The email subject (supports {{variable}} placeholders)
  • HTML Body — The email content in HTML format
  • SMTP Credentials — Server connection details (stored encrypted)

Unlayer Drag-and-Drop Editor

For rich email designs, click the Unlayer Editor button when editing a template. The Unlayer editor provides:

  • Drag-and-drop content blocks (text, images, buttons, dividers)
  • Responsive design that works on all devices
  • Pre-built layouts and templates
  • HTML export that saves directly to your template

Test Email

After creating a template, use the Send Test Email button to verify your SMTP settings. Enter a recipient address and a test email will be sent using the template's configuration.

Per-Agent Email Toggles

To enable email sending for an incoming agent:

  1. Edit the agent and scroll to System Tools
  2. Check Email Sending
  3. Select the email template to use
  4. Save the agent — the system automatically creates a send_email webhook tool in ElevenLabs

WhatsWay Setup

WhatsWay provides a simple API gateway for sending WhatsApp messages without requiring Meta Business verification.

Prerequisites

  • A WhatsWay account at whatsway.com
  • An active WhatsWay instance with a connected phone number
  • Your API Key and API Secret from the WhatsWay dashboard

Configuration

Enter Credentials

Go to Settings → Messaging → WhatsApp. Enter your WhatsWay API Key and API Secret. Optionally configure Base URL and Channel ID.

Test Connection

Click Test Connection to verify your credentials are valid and the instance is active.

Save Settings

Click Save. The button will show “Saved!” with a checkmark on success.

Template Messages

WhatsWay templates are fetched from your WhatsWay account. To use template messages:

  1. Create templates in the WhatsWay dashboard
  2. Templates appear automatically in the Templates section of your messaging settings
  3. Select a template when configuring your agent's WhatsApp tool
⚠
Note: WhatsWay supports outbound messaging only. For inbound message handling and a conversations inbox, use the Meta WhatsApp Business API integration.

Meta WhatsApp Business API Setup

The Meta integration uses the official WhatsApp Cloud API, providing full-featured messaging with inbound webhooks, template messages, media support, and the conversations inbox.

Prerequisites

  • A Meta Developer account at developers.facebook.com
  • A Meta App with WhatsApp product enabled
  • Your Meta App ID and App Secret

Admin Configuration

The admin must configure the Meta App credentials before users can connect their WhatsApp Business accounts.

Set Provider Mode

Go to Admin Panel → Settings → Messaging. Set the WhatsApp provider mode to Meta or Both (WhatsWay + Meta).

Enter Meta App Credentials

Enter your Meta App ID and Meta App Secret from the Meta Developer dashboard.

Generate Verify Token

Click Generate Verify Token to create a random token used for Meta webhook verification.

Test Connection

Click Test Meta App Connection to validate your App ID and App Secret against the Meta Graph API.

Configure Meta Webhook

Copy the Webhook URL shown in the admin panel and paste it into your Meta App's WhatsApp webhook configuration at developers.facebook.com. Use the generated Verify Token as the verification token.

ℹ
Webhook URL Format: https://yourdomain.com/api/webhooks/messaging/meta/webhook

User Setup — Embedded Signup

Once the admin has configured Meta credentials, users can connect their WhatsApp Business accounts through Meta's Embedded Signup flow:

  1. Go to Settings → Messaging → WhatsApp
  2. Click Connect with Facebook
  3. Log in with your Facebook account and follow the prompts
  4. Select or create a WhatsApp Business Account
  5. Your WABA ID, Phone Number ID, and Access Token are saved automatically

User Setup — Manual Entry

Users can also manually enter their WABA credentials:

Field Description Where to Find
WABA ID WhatsApp Business Account ID Meta Business Manager → WhatsApp Accounts
Phone Number ID The phone number resource ID Meta Developer Dashboard → WhatsApp → API Setup
Access Token Permanent or System User access token Meta Business Manager → System Users

Template Messages

Meta WhatsApp templates are managed in Meta Business Manager. Once approved by Meta, they appear automatically in your template list. Templates are required for initiating conversations outside the 24-hour messaging window.

Inbound Messages

When a customer sends a message to your WhatsApp number, the Meta webhook delivers it to AgentLabs. Messages appear in the Conversations Inbox in real time.

Conversations Inbox

The Conversations Inbox provides a full-featured WhatsApp messaging interface for managing customer conversations.

Accessing the Inbox

Navigate to Settings → Messaging → Conversations tab. The inbox shows all WhatsApp conversations associated with your connected number.

Inbox Features

  • Conversation List — All conversations sorted by most recent message, with unread count badges
  • Real-Time Updates — New messages appear automatically via polling
  • Send Messages — Reply to customers with text messages
  • Media Support — Send and receive images, documents, and other media files
  • Read Receipts — Mark conversations as read
  • Conversation Status — Update conversation status (open, resolved, etc.)
  • Contact Information — View customer name and phone number
ℹ
Note: The Conversations Inbox is only available with the Meta WhatsApp Business API integration. WhatsWay does not support inbound message handling.

24-Hour Messaging Window

Meta's policy allows free-form messaging only within 24 hours of the customer's last message. After 24 hours, you must use an approved template message to re-initiate the conversation.

Admin Settings

The admin messaging panel provides centralized configuration and oversight for the entire messaging system.

Accessing Admin Settings

Navigate to Admin Panel → Settings → Messaging.

Provider Mode

Control which WhatsApp provider is available to users:

Mode Description
WhatsWay Only Users can only configure WhatsWay for WhatsApp messaging
Meta Only Users can only use Meta WhatsApp Business API (requires admin Meta config)
Both Users can choose either WhatsWay or Meta WhatsApp

Meta Configuration

When Meta is enabled, the admin panel shows:

  • Meta App ID — Your Meta Developer App ID
  • Meta App Secret — Your Meta Developer App Secret (masked for security)
  • Verify Token — Auto-generated token for Meta webhook verification
  • Webhook URL — The URL to configure in Meta Developer Dashboard
  • Test Meta App Connection — Validates your App credentials against the Meta Graph API

Statistics Dashboard

The admin dashboard shows aggregate messaging statistics:

  • Total messages sent (email + WhatsApp)
  • Total email templates across all users
  • Total WhatsApp configurations
  • Active conversations count

Messaging Logs

Admin can view all messaging logs across all users, including:

  • Sender and recipient details
  • Channel type (email / WhatsApp)
  • Delivery status (success / failed)
  • Timestamps and error messages

Webhook Reference

The messaging plugin uses webhooks for two purposes: receiving AI agent tool calls from ElevenLabs, and receiving inbound WhatsApp messages from Meta.

ElevenLabs Agent Webhook Tools

When an incoming agent has email or WhatsApp enabled, the system automatically creates webhook tools that ElevenLabs calls during live conversations:

POST /api/webhooks/messaging/send-email/:token/:agentId

Called by ElevenLabs when the AI agent decides to send an email. The :token is a per-user webhook authentication token. Body includes recipient email and variable values extracted from the conversation.

POST /api/webhooks/messaging/send-whatsapp/:token/:agentId

Called by ElevenLabs when the AI agent decides to send a WhatsApp message. Uses the user's configured WhatsApp provider (WhatsWay or Meta) to deliver the message.

ℹ
Webhook Token: Each user has a unique webhook token that authenticates ElevenLabs webhook calls. This token is generated automatically and prevents unauthorized access to the messaging endpoints.

Meta WhatsApp Inbound Webhook

GET /api/webhooks/messaging/meta/webhook

Webhook Verification. Meta sends a GET request with hub.mode, hub.verify_token, and hub.challenge query parameters. The endpoint validates the verify token and echoes back the challenge.

POST /api/webhooks/messaging/meta/webhook

Inbound Messages. Meta sends incoming WhatsApp messages, delivery status updates, and other events to this endpoint. The plugin validates the X-Hub-Signature-256 header using the Meta App Secret, then processes the payload.

Webhook Security

All webhook endpoints are secured:

  • ElevenLabs webhooks — Authenticated via the :token URL parameter (unique per user)
  • Meta webhooks — Verified via X-Hub-Signature-256 HMAC signature using the Meta App Secret
  • Meta verification — Uses the admin-configured verifyToken to validate webhook subscription

Security Best Practices

Credential Storage

  • SMTP passwords are stored in the database alongside templates — ensure your database is encrypted at rest
  • WhatsWay API keys and Meta access tokens are stored per-user in the database
  • Meta App Secret is stored in the admin configuration table
  • All sensitive fields are masked in API responses (only last 4 characters shown)

Webhook Token Validation

Each user has a unique webhook token generated from their user ID and a server-side secret. The token is validated on every webhook call using HMAC-SHA256, ensuring only legitimate ElevenLabs requests can trigger message sending.

Meta Webhook Signature Verification

Inbound Meta webhooks include an X-Hub-Signature-256 header containing a HMAC-SHA256 hash of the request body. The plugin verifies this signature against the stored Meta App Secret before processing any inbound message.

API Authentication

  • All user-facing endpoints require session-based authentication
  • Admin endpoints require admin-level session authentication
  • Webhook endpoints use token-based authentication (no session required)

Transport Security

  • Always deploy AgentLabs behind HTTPS — Meta requires HTTPS for webhook URLs
  • SMTP connections use TLS/STARTTLS when the port supports it (587 for STARTTLS, 465 for implicit TLS)
  • Meta Graph API calls use the Authorization: Bearer header (credentials are never placed in URL query strings)
⚠
Important: Never share your Meta App Secret, SMTP passwords, or webhook tokens. If you suspect any credentials have been compromised, rotate them immediately from the respective provider dashboards.

Troubleshooting

SMTP Connection Failed

  • Verify your SMTP host and port are correct (common: port 587 for TLS, 465 for SSL)
  • For Gmail, use an App Password instead of your regular password
  • Check if your SMTP provider requires enabling "Less Secure Apps" or similar settings
  • Ensure your server can reach the SMTP host (firewall rules, DNS resolution)

WhatsWay Connection Failed

  • Verify your API Key and API Secret are correct
  • Check that your WhatsWay instance is active and the phone number is connected
  • Ensure your WhatsWay subscription is active

Meta Webhook Verification Failed

  • Ensure the Verify Token in the admin panel matches exactly what you entered in the Meta Developer Dashboard
  • Verify the webhook URL is publicly accessible over HTTPS
  • Check that the webhook URL path is correct: /api/webhooks/messaging/meta/webhook

Meta Test Connection Failed

  • Double-check your Meta App ID and App Secret from the Meta Developer Dashboard
  • Ensure the WhatsApp product is added to your Meta App
  • Verify the app is not in "development" mode if you need production access

Embedded Signup Not Working

  • Ensure the admin has configured Meta App ID in the admin panel
  • The Meta App must have the WhatsApp Embedded Signup product enabled
  • Check browser console for any JavaScript errors from the Facebook SDK
  • Ensure your domain is added to the Meta App's allowed domains

Messages Not Sending from Agent

  • Verify the agent has email/WhatsApp enabled under System Tools
  • Check that a template is selected for the agent
  • Review the messaging logs for error details
  • Ensure the user's WhatsApp/SMTP credentials are valid

Conversations Not Appearing

  • Conversations only work with Meta WhatsApp (not WhatsWay)
  • Verify your Meta webhook is correctly configured and receiving events
  • Check the server logs for webhook processing errors

Plugin Not Loading

  • Verify the plugin files are in plugins/messaging/
  • Check that plugin.json exists and is valid JSON
  • Ensure all database migrations have been run
  • Restart the AgentLabs server after installation
  • Check server logs for plugin loading errors
